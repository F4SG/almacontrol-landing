2. REQUERIMIENTOS TÉCNICOS
================================================================

STACK TECNOLÓGICO
---------------------------------
✅ Opción elegida: React (Framework por separado)
   - HTML: generado por React desde JSX (index.html + componentes)
   - CSS: Tailwind CSS v4 con clases utilitarias
   - JavaScript: ES2022+ con React 18 (hooks: useState, useEffect)
   - Framework: React 18 + Vite 8 como build tool
   - Íconos: lucide-react

   Archivos principales:
   - index.html        → entrada HTML
   - src/index.css     → estilos globales + Tailwind
   - src/main.jsx      → punto de entrada JavaScript
   - src/App.jsx       → componente raíz
   - src/components/   → 9 componentes React separados


FORMULARIO DE CAPTURA — VALIDACIONES CLIENT-SIDE
---------------------------------
✅ e.preventDefault() → No recarga la página al enviar

✅ Campos obligatorios con mensajes de error en rojo:
   - Nombre vacío → "El nombre es requerido"
   - Email vacío  → "El correo es requerido"
   - Email con formato inválido → "Ingresa un correo válido"
   - Empresa vacía → "El nombre de empresa es requerido"
   - Tamaño sin seleccionar → "Selecciona el tamaño de empresa"

✅ Validación de email con expresión regular (regex):
   /^[^\s@]+@[^\s@]+\.[^\s@]+$/

✅ Al enviar exitosamente:
   - Se previene la recarga (e.preventDefault)
   - Se limpian todos los campos del formulario
   - Se muestra el Modal de agradecimiento

✅ Modal de agradecimiento con imagen:
   - Imagen de almacén como cabecera del modal
   - Ícono de check verde sobre la imagen
   - Título: "¡Registro exitoso!"
   - Mensaje: "Te contactaremos en las próximas 24 horas..."
   - Cierra con: botón X, botón "Cerrar" o clic fuera del modal
   - Cierra también con la tecla Escape
   → Archivo: src/components/Modal.jsx


DISEÑO RESPONSIVO — MOBILE FIRST
---------------------------------
✅ Mobile First implementado con Tailwind CSS:
   - Base (mobile, 375px): 1 columna, menú hamburguesa
   - sm: (640px tablet): 2 columnas en formulario y métricas
   - md: (768px): 3 columnas en pricing y testimonios
   - lg: (1024px desktop): 2 columnas hero, 3 features, 4 footer

✅ Navbar responsiva:
   - Desktop: links horizontales + botón CTA
   - Mobile: menú hamburguesa con toggle (useState)

✅ Imágenes responsivas: object-cover con altura fija adaptable


BUENAS PRÁCTICAS
---------------------------------
✅ Código limpio y estructurado:
   - 9 componentes separados, cada uno en su propio archivo
   - Sin lógica mezclada entre componentes
   - Datos separados de la lógica (arrays de objetos al inicio)

✅ Uso semántico de etiquetas HTML:
   - <nav> para la barra de navegación
   - <section> para cada sección con id único
   - <main> para el contenido principal
   - <footer> para el pie de página
   - <form>, <label>, <input>, <select> para el formulario
   - <ul>, <li> para listas de features y planes
   - <h1>, <h2>, <h3> con jerarquía correcta

✅ Sin errores en consola del navegador:
   - Keys únicas en todas las listas (.map con key={id})
   - Props válidas y tipadas correctamente
   - Sin variables no declaradas ni imports faltantes
